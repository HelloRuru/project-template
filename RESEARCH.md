# 遊戲研究筆記：宮鬥世代傳承遊戲

> 調查日期：2026-02-23
> 遊戲方向：類《甄嬛傳》宮鬥遊戲，世代傳承，從甄選入宮開始
> 玩家角色：女性（嬪妃視角），不可選男性

---

## 遊戲概念

玩家扮演一名即將入宮甄選的女子，從甄選那一刻起就開始宮鬥。透過智謀、人際關係、才藝培養，一步步在後宮中生存與晉升。核心特色是**世代傳承**——培養子嗣，讓家族跨越世代延續權力與榮耀。

---

## 市場競品分析

### 1. 深宮曲（最值得參考）

- **平台**：易次元 App（文字互動作品）
- **類型**：高自由度文字養成
- **玩法**：
  - 自由選擇出身（貴族或平民）
  - 可選角色：初入宮秀女 / 冷宮棄妃
  - 每日行動：練琴、練舞、讀書、刺繡
  - 賄賂太監、爭取皇帝臨幸
  - 結交太后或高位嬪妃抬升位份
  - 造謠、下毒等陰謀手段
  - 接濟娘家助長家族勢力
- **世代傳承**：培養皇子 → 立為太子 → 投毒皇帝 → 兒子登基 → 成為太后垂簾聽政
- **為何值得參考**：自由度極高、文字為主、有世代傳承機制、宮鬥元素最完整

### 2. 熹妃傳

- **平台**：iOS / Android
- **類型**：宮鬥養成手遊
- **標語**：「第一部可以玩的宮鬥小說」
- **玩法**：
  - 穿越成為經典大劇女主角
  - 勾心鬥角拼官權
  - 打理府邸、養小孩
  - 培養孩子成為天子
  - 裝潢佈置豪宅
  - 超開放結局
- **缺點**：課金門檻高，不花台幣很難完成活動
- **為何值得參考**：養小孩 + 當太后的世代傳承概念

### 3. 熹妃 Q 傳

- **平台**：iOS / Android
- **類型**：Q 版 3D 宮廷交友手遊
- **玩法**：
  - 入宮選秀成為皇上的女人
  - 劇情對話自由選擇，影響結局
  - 自由穿搭、精緻妝容
  - 皇子養成
  - 種菜釣魚田園生活
  - 跨服選美
  - 戀愛結婚系統（戀愛八級可求婚）
- **為何值得參考**：多結局系統、皇子養成

### 4. 后宮甄嬛傳（官方授權手遊）

- **平台**：iOS / Android
- **開發**：藍港互動（流瀲紫唯一授權）
- **類型**：3D 卡牌收集手遊
- **玩法**：
  - 卡牌收集近 200 張精美卡牌
  - 三大陣營：華妃、甄嬛、皇后
  - 電視劇原班人馬配音
  - 萌寵助攻對戰系統
- **為何值得參考**：IP 授權的成功案例、陣營對立設計

### 5. 後宮三千人

- **平台**：iOS / Android
- **類型**：互動小說
- **玩法**：
  - 靠對話選擇影響劇情發展
  - 多種結局
  - 像看古風言情小說，代入感極強
  - 可以挑戰當上皇后
- **為何值得參考**：純文字 + 選擇驅動，最接近我們想做的形式

### 6. 宮廷計

- **平台**：iOS / Android
- **類型**：古風宮廷養成手遊
- **玩法**：東方玄幻色彩，側福晉進宮成為嫻妃，後宮生存
- **為何值得參考**：宮廷背景設定、位份晉升系統

### 7. Road to Empress（成世天下）

- **平台**：手機互動遊戲
- **來源**：中國製作，在韓國爆紅
- **玩法**：
  - 分支敘事結構
  - 玩家選擇影響故事走向
  - 宮廷權謀與愛情交織
- **為何值得參考**：跨國成功案例、互動敘事設計

### 8. 橙光平台

- **平台**：iOS / Android / Web
- **類型**：互動小說創作平台
- **規模**：3000 萬註冊用戶、170 萬投遞作品
- **宮鬥作品**：大量宮鬥、宮廷權謀類互動小說
- **注意**：現需中國手機實名認證，海外用戶使用困難
- **為何值得參考**：平台模式、大量宮鬥作品的設計模式可借鏡

### 9. 其他相關遊戲

|------|------|

## 世代傳承機制分析

### 目前市面上的做法

| 遊戲 | 傳承方式 |
|------|---------|
| 深宮曲 | 培養皇子 → 立太子 → 當太后垂簾聽政 |
| 熹妃傳 | 培養孩子成為全能天子，自己當太后 |
| 熹妃 Q 傳 | 皇子養成系統 |
| CK3 | 角色死亡後切換到繼承人，保留家族資產與聲望 |

### 我們的傳承設計方向

1. **第一代**：玩家從甄選入宮，目標是生存、晉升、生下子嗣
2. **第二代**：以女兒或媳婦的視角繼續遊戲，繼承母親的人脈與資源
3. **第三代以後**：家族累積的聲望、財富、秘辛成為新的遊戲資源
4. **跨世代要素**：
   - 家族聲望值（跨代累積）
   - 傳承技能（如琴藝、毒術、醫術）
   - 家族秘辛（可作為把柄或護身符）
   - 宮廷人脈網路
   - 歷代恩怨關係影響新一代的起始處境

## 核心系統規劃

### A. 甄選入宮系統（遊戲開局）

- 角色創建：家世背景、容貌、才藝、個性
- 甄選過程：多輪篩選，每輪都可能被淘汰
- 初始陣營：根據出身自動劃定初始關係網
- 首次分配：被分配到哪個宮殿，影響初期人際關係

### B. 位份晉升系統

| 位份 | 說明 |
|------|------|
| 答應 | 最低位份，剛入宮 |
| 常在 | 略有寵幸 |
| 貴人 | 開始有自己的宮女 |
| 嬪 | 可參與後宮大事 |
| 妃 | 掌管一宮事務 |
| 貴妃 | 後宮高位，權傾一方 |
| 皇貴妃 | 僅次於皇后 |
| 皇后 | 後宮之主 |
| 太后 | 終極目標，垂簾聽政 |

### C. 屬性系統

| 屬性 | 影響 |
|------|------|
| 容貌 | 皇帝寵幸機率、甄選表現 |
| 才藝 | 琴棋書畫，提升文化聲望 |
| 心計 | 識破陰謀、策劃行動成功率 |
| 體質 | 懷孕機率、抗毒能力、壽命 |
| 人緣 | 宮女忠誠、太監合作、妃嬪結盟 |
| 福運 | 隨機事件的正面傾向 |

### D. 行動系統（每日 / 每旬）

- **日常行動**：練琴、讀書、刺繡、散步、禮佛
- **社交行動**：拜訪其他嬪妃、結盟、挑撥離間
- **陰謀行動**：下毒、造謠、賄賂太監、安插眼線
- **皇帝相關**：爭寵、侍寢、同遊、進言
- **家族行動**：接濟娘家、聯姻佈局、召見家人
- **子女行動**：教育子女、安排聯姻、培養繼承人

### E. 人際關係系統

- **皇帝好感度**：影響寵幸頻率、位份晉升
- **太后好感度**：影響後宮地位、保護傘效果
- **嬪妃關係**：結盟 / 中立 / 敵對
- **宮女忠誠度**：影響情報蒐集、陰謀執行
- **太監關係**：影響消息通報、辦事效率
- **娘家關係**：影響外朝支援、家族勢力

### F. 子嗣系統

- **懷孕**：受體質、皇帝寵幸頻率、後宮安全影響
- **保胎**：防範其他嬪妃的暗算
- **生產**：男女隨機，影響後續策略
- **教育培養**：決定子女屬性成長方向
- **聯姻**：與朝臣家族聯姻，擴展勢力
- **傳承觸發**：玩家角色年老 / 駕崩 → 切換到下一代

### G. 自動存檔觸發點

- 進入新關卡 / 章節前
- 做出重大決定前（結盟、背叛、下毒等）
- 懷孕 / 生產事件
- 位份變動
- 世代傳承切換前
- 每日遊戲自動存檔（防當機）

## 技術架構建議

### 推薦方案

```
前端：Next.js (App Router) + Tailwind CSS
後端/驗證：Supabase (Auth + PostgreSQL + Realtime)
部署：Cloudflare Pages
```

### 為何選這套

- **帳號密碼登入**：Supabase Auth 內建 Email/密碼驗證
- **雲端存檔**：Supabase PostgreSQL 儲存遊戲狀態
- **跨裝置**：登入後自動載入最新存檔
- **自動存檔**：透過 debounce 機制，重要操作後即時寫入
- **離線保護**：localStorage 暫存 + 上線後同步
- **響應式**：Tailwind CSS + Mobile-first 設計
- **部署**：符合現有 Cloudflare Pages 工作流

### 存檔機制設計

```
1. 自動存檔：每次重要操作後寫入 Supabase
2. 防抖存檔：debounce 2-3 秒，避免頻繁寫入
3. 離線快取：localStorage 暫存，恢復網路後同步
4. 衝突處理：以伺服器端時間戳為準
5. 多存檔位：保留最近 5 個存檔點 + 1 個自動存檔
6. 關鍵存檔：進關卡、做重大決定前強制存檔
```
